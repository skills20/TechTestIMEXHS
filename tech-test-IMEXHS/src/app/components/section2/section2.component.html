High-Level Architecture:

  1. Authentication Module: Responsible for user authentication, ensuring only radiologists can access the system.
  2. Study Management Module: Handles the storage and retrieval of radiological study information.
  3. Questionnaire Module: Manages the questionnaire for radiologists to fill out about each study.
  4. Statistics Module: Provides endpoints for HIS to retrieve statistics about studies read by radiologists.
  5. Data Encryption Module: Ensures sensitive patient information is encrypted.
  6. Communication Module: Responsible for communicating with the HIS for sending study results.

Information Flows:

  1. Receiving New Radiological Study:
    • When the HIS sends a JSON message, the system receives it and stores the information.
    • Patient information (e.g., name, age) is encrypted before storage.
    • Study details are stored as they are.
  2. Radiologist Workflow:
    • Radiologists authenticate to access the system.
    • They see a list of pending studies and choose one.
    • They answer the questionnaire, which includes questions about water volume and findings.
    • Responses, along with the date and the radiologist's user ID, are stored in the system.
  3. Reading Study Submission:
    • When a radiologist submits a form, the system sends a POST request to HIS with the necessary data, including an authorization header.
  4. Statistics:
    • HIS can make GET requests to the statistics endpoint with specific parameters.
    • The system processes the requests and returns statistics in JSON format.

Module Decomposition:

  1. Authentication Module:
    • Responsible for user authentication.
    • Ensures security measures are in place.
  2. Study Management Module:
    • Handles the storage and retrieval of study information.
    • Encrypts patient data before storage.
  3. Questionnaire Module:
    • Manages the questionnaire for radiologists.
    • Calculates the water percentage and displays it to radiologists.
  4. Statistics Module:
    • Provides endpoints for HIS to retrieve statistics.
    • Handles statistics calculation based on parameters.
  5. Data Encryption Module:
    • Responsible for encrypting sensitive patient information.
  6. Communication Module:
    • Manages communication with HIS for sending study results.
    • Includes authorization headers.
